调研阶段做了什么

第一步：理解仓库
读 README、扫目录结构、获取 GitHub 统计数据（stars、forks、issues）。搞清楚这个项目解决什么问题、用什么方法。
第二步：搜真实用户反馈
在 X/Twitter、Reddit、Hacker News、DEV.to 上搜索这个仓库的名字。找到真实用户的评价和使用经验，优先选浏览量高的帖子。
第三步：找直接竞品
确认这个仓库解决什么问题，然后找解决同一问题的其他独立开源项目。不用内置功能或 DIY 拼凑方案凑数。
第四步：生成指南
把调研结果组织成 7 个模块，用 9 种互动元素呈现。输出一个自包含的 HTML 文件，离线也能看（除了 Google Fonts）。
?
检查一下你的理解
GitHub Guide 在生成指南之前，会先做什么？
正确！GitHub Guide 会在 X/Twitter、Reddit、Hacker News、DEV.to 等平台搜索这个仓库的真实用户评价，优先选浏览量高的帖子。每条引用都有原文链接，不会编造。如果找不到真实反馈，会诚实说明。

04

跟其他工具有什么不同？

市面上也有把仓库变成文档/教程的工具

三种思路

📝
思路 A：生成 README

readme-ai 这类工具用 AI 自动生成 Markdown README 文件。输出是纯文字，没有交互元素，适合开发者快速写文档。

🌐
思路 B：在线 Wiki

DeepWiki 这类 SaaS 工具把仓库变成在线 Wiki 页面。功能强大但偏技术向，需要联网访问，面向开发者而非小白。

🎯
思路 C：GitHub Guide

小白友好的交互式 HTML。自动调研真实用户反馈，用动画和测验代替大段文字，单文件离线可看。
