What happens during research

Step 1: Understand the repo
Read the README, scan directory structure, check GitHub stats (stars, forks, issues). Identify what problem it solves and its unique approach.
Step 2: Find real user feedback
Search X/Twitter, Reddit, Hacker News, and DEV.to for the repo name. Prioritize high-engagement posts (>10K views). Every case must have a source link.
Step 3: Find direct competitors
Identify the core problem, then find other open-source projects solving the same problem. No built-in features or DIY workarounds — only standalone projects.
Step 4: Generate the guide
Organize everything into 7 modules with 9 interactive element types. Output a single self-contained HTML file that works offline (except Google Fonts).

How does GitHub Guide select competitors for the comparison table?
Correct! GitHub Guide first identifies the core problem the repo solves, then searches for other standalone open-source projects tackling the same problem. It won't include built-in features of other products or DIY workaround combinations.
