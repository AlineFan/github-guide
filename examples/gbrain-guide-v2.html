为什么 AI 需要"记忆"？

你有没有发现，每次跟 AI 新对话，它都不记得你之前说了什么？

每天都失忆的超级助理

想象你雇了一个超厉害的助理 — 他什么都会，但有个致命缺陷：每天上班都失忆。

你昨天跟他说了半天的项目背景，今天他全忘了。你每次都要从头解释。很崩溃对吧？

这就是现在 AI 的状态。

GBrain 做了什么？

GBrain 就是给你的 AI 助理配了一本「你的人生百科全书」。它把你的各种信息汇总到一个智能数据库里，AI 随时能翻阅、搜索、更新。

📧
邮件和日历

自动导入 Gmail、Google Calendar，AI 知道你什么时候有会，跟谁聊了什么。

📝
笔记和想法

从 Obsidian、Notion、Logseq 导入笔记，AI 能直接利用你积累的知识。

🔄
越用越聪明

每次和 AI 对话都会丰富知识库。用得越多，AI 对你的理解就越深。

💡
核心理念

GBrain 的口号是："Repo 是事实来源，GBrain 是检索层，Agent 通过两者读写。人类永远优先 — 你可以直接编辑任何 Markdown 文件。"

智能搜索：不只是关键词匹配

普通的搜索引擎只能做关键词匹配 — 你搜"开会"，它只找包含"开会"两个字的文档。

GBrain 用的是混合搜索 — 它能理解"意思"。搜"上次跟客户聊了什么"，即使你的笔记里没有"客户"这个词，它也能找到相关内容。

🔤
普通搜索

像 Ctrl+F 一样，只能找到完全匹配的关键词。搜"苹果"就只找"苹果"，不会找到"iPhone"。

🧠
GBrain 智能搜索

关键词 + 理解意思，双重匹配。搜"苹果"能同时找到水果和苹果公司的内容，还能根据你的上下文判断你想找哪个。

检查一下你的理解
GBrain 的"读写循环"和普通 RAG 检索最大的区别是什么？
正确！ 普通的 RAG 只能"读"— AI 从文档库里找信息，但不会把新信息写回去。GBrain 的核心创新是"读写循环"：每次对话都会让知识库变得更丰富。就像一个越用越聪明的笔记本。

跟其他工具有什么不同？

市面上也有类似的"AI 记忆"方案，但设计思路各不相同

解决的核心问题

"AI 没有长期记忆"这个问题，目前有三种主流解决思路：

🔍
思路 A：云端记忆 API

Mem0、Letta 这类工具提供云端 API，方便集成但数据存在别人服务器上。生态大、功能强，适合企业和 Agent 开发者。

🖥️
思路 B：自托管助手

Khoj 这类工具可以自己部署，有 Web 界面，上手简单。但搜索能力和 AI 写入能力相对有限。

🧠
思路 C：GBrain 的方式

本地优先 + 读写循环。数据在你电脑上，AI 不仅能搜索还能主动记录和整理。知识像存款一样复利增长。

最简单的上手方式

三步搞定，不到 5 分钟

前提条件

你需要先安装好这些工具（如果你用 Mac 开发，大概率已经有了）：

📦
Bun

JavaScript 运行时和包管理器。安装：curl -fsSL https://bun.sh/install | bash

🐙
Git

版本控制工具。Mac 自带，Windows 去 git-scm.com 下载。

三步开始

1
下载并安装

把代码下载到你的电脑，安装依赖包。

TERMINAL # 下载 GBrain 到你的 home 目录 git clone https://github.com/garrytan/gbrain.git ~/gbrain # 进入目录，安装依赖 cd ~/gbrain && bun install && bun link
2
初始化数据库

一条命令创建本地数据库，不需要装 PostgreSQL，不需要注册任何账号。

TERMINAL # 初始化 — 自动创建嵌入式本地数据库 gbrain init
3
导入内容 & 开始提问

把你的笔记文件夹导入，然后就可以开始跟你的知识库对话了。

TERMINAL # 导入你的笔记（支持 Markdown 文件） gbrain import ~/你的笔记文件夹/ # 生成向量索引（需要 OpenAI API Key） gbrain embed --stale # 开始提问！ gbrain query "我最近在研究什么？"
💡
不想用 OpenAI？

向量嵌入需要 OpenAI API Key（少量费用）。如果你暂时不想用，可以先运行 gbrain import --no-embed 只导入不生成向量，之后再补。

进阶：接入 Claude Code

如果你用 Claude Code 写代码，可以通过 MCP Server 让 Claude 直接读写你的知识库。

CLAUDE CODE 配置 # 在 Claude Code 的 MCP 设置中添加： { "mcpServers": { "gbrain": { "command": "gbrain", "args": ["mcp"] } } }

配好后，Claude Code 就能使用 search、get_page、put_page 等 30+ 个工具来读写你的知识库。

🎉
就这么简单！

从 clone 到第一次提问，整个过程不到 5 分钟。之后每次 AI 对话都会让你的知识库变得更丰富。
